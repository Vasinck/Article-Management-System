# 文章提取工作流文档

本文档详细记录了从 `new` 文件夹提取 HTML 文章内容并写入 `article_data.json` 的完整工作流。

## 🔄 完整工作流

### 第一步：检查 new 目录中的 HTML 文件

**操作：** 列出 `new` 目录中所有待处理的 HTML 文件

**命令：**
```bash
dir "E:\CDL\work\项目\文章记录器\new\*.html" /b
```

**输出示例：**
```
CVPR _ 比 Dynamic Conv 更高效，比 Attention 更轻量：TVConv 为何更适合结构化视觉任务？.html
CVPR2026 _ 告别慢推理！腾讯全新检索式目标检测模型 WeDetect，15 个基准霸榜还快 13 倍.html
FPN 家族再添大将 FGAA-FPN _ 90！.html
```

**目的：** 确认有多少篇新文章需要处理，获取准确的文件名列表

---

### 第二步：读取 HTML 文件并提取信息

**操作：** 读取每个 HTML 文件，提取标题和正文内容

**方法 1：使用 read_file 工具直接读取**
```
read_file <文件绝对路径>
```

**方法 2：使用 task 工具委托子代理批量处理**
```
task: "读取新文章生成关键句"
subagent_type: "general-purpose"
prompt: "请读取 E:\CDL\work\项目\文章记录器\new 目录下的以下 X 个 HTML 文件..."
```

**关键提取点：**
1. **标题**：从 `<meta property="og:title" content="xxx">` 标签提取
2. **描述**：从 `<meta name="description" content="xxx">` 或 `<meta property="og:description">` 提取
3. **作者**：从 `<meta name="author" content="xxx">` 提取
4. **正文**：从 `js_content` 容器或文章主体区域提取

### 第三步：生成关键句（tags 字段）

**关键句格式要求：**

| 要求 | 说明 |
|------|------|
| 数量 | 2-4 句 |
| 第一句 | 说明领域，格式为"XXX 领域的 XXX 论文" |
| 其余句 | 只讲技术细节，不含实验数据 |
| 分隔符 | 使用分号 `；` 分隔 |

**领域判断参考表：**

| 关键词 | 领域标识 |
|--------|----------|
| YOLO/检测/目标 | 目标检测 |
| DETR/匈牙利算法 | 目标检测/DETR 训练优化 |
| 注意力/Attention | 注意力机制 |
| Transformer/ViT | 视觉 Transformer |
| LoRA/微调/参数 | 大语言模型/参数高效微调 |
| 蒸馏/知识蒸馏 | 知识蒸馏 |
| 分割/语义分割 | 图像分割 |
| 跟踪/追踪 | 视觉目标跟踪 |

**关键句生成示例：**

```
目标检测/YOLO 架构论文；基于 YOLOv12 架构，引入区域注意力 C2f 模块来增强极小目标的特征提取；集成 C3K2 模块以提升检测框架的整体效率
```

```

```

### 第四步：去重检查

**操作：** 检查是否有重复文章

**判断依据：**
- 标题相似度 > 90%
- 描述内容基本一致
- 同一研究的不同报道版本

**处理方式：** 保留一篇，删除重复项

---

### 第五步：生成文章条目

**JSON 格式：**
```json
{
    "id": "8 位随机十六进制字符串",
    "title": "文章标题",
    "tags": ["关键句 1；关键句 2；关键句 3"]
}
```

**ID 生成方法：**
```python
import random
import string
article_id = ''.join(random.choices(string.hexdigits.lower(), k=8))
# 示例输出：a8f3c2d1
```

### 第六步：写入 article_data.json

**完整 Python 脚本：**

```python
#!/usr/bin/env python
# -*- coding: utf-8 -*-
import json
import random
import string

# 读取现有数据
with open(r'E:\CDL\work\项目\文章记录器\article_data.json', 'r', encoding='utf-8') as f:
    data = json.load(f)

# 新文章列表
new_articles = [
    {
        'id': ''.join(random.choices(string.hexdigits.lower(), k=8)),
        'title': '文章标题',
        'tags': ['关键句 1；关键句 2；关键句 3']
    },
    # ... 更多文章
]

# 添加新文章
data['articles'].extend(new_articles)

# 写回文件
with open(r'E:\CDL\work\项目\文章记录器\article_data.json', 'w', encoding='utf-8') as f:
    json.dump(data, f, ensure_ascii=False, indent=4)

print(f'成功添加 {len(new_articles)} 篇文章')
```

**编码安全要求（必须遵守）：**
- 不要通过 PowerShell 管道、here-string 或标准输入把**含中文内容的 Python 脚本**直接传给 `python -`
- 这类写法在 Windows 控制台编码不一致时，可能会在进入 Python 前就把中文替换成 `?`
- 正确做法是：先创建 `add_articles.py` 文件，再执行 `python add_articles.py`
- 如果只是少量修改 JSON，也可以直接编辑文件，但写回时必须保持 UTF-8 编码
- 写入完成后，必须重新用 `encoding='utf-8'` 读取一次 `article_data.json`，确认新增条目的 `title` 和 `tags` 没有出现 `?`

**执行命令：**
```bash
python add_articles.py
```

**清理临时文件：**
```bash
del add_articles.py
```

## ⚠️ 注意事项

### 1. 文件路径问题
- 必须使用**绝对路径**
- 路径中包含中文时，确保 Python 脚本使用 `encoding='utf-8'`
- Windows 路径使用原始字符串 `r'path'` 或双反斜杠 `\\`

### 2. HTML 文件编码
- 微信公众号 HTML 通常使用 UTF-8 编码
- 读取时添加 `errors='ignore'` 处理特殊字符

### 3. JSON 格式
- 使用 `ensure_ascii=False` 保留中文字符
- 使用 `indent=4` 保持格式美观
- 不要使用 `python -` 配合 PowerShell 管道执行含中文脚本，否则中文可能被写成 `?`
- 最稳妥的方式是将脚本保存为 `.py` 文件后再执行
- 写入后务必重新读取 JSON 并检查新增条目的中文字段是否正常

### 4. 去重逻辑
- 同一研究的不同报道版本视为重复
- 保留标题更完整/描述更准确的一篇

### 5. 关键句质量
- 第一句必须明确领域
- 技术细节要具体，避免空泛描述
- 不包含实验数据（如"提升 5.8%"、"达到 86.0% 准确率"）

## 🔧 常见问题解决

### 问题 1：HTML 文件无法读取
**原因：** 文件名包含特殊字符或路径错误
**解决：** 使用 `dir /b` 命令获取准确文件名，复制粘贴到脚本中

### 问题 2：关键句生成不完整
**原因：** HTML 正文内容提取失败
**解决：** 基于标题和描述信息，结合领域知识推断生成

### 问题 3：JSON 写入失败
**原因：** 文件被占用或编码问题
**解决：** 关闭其他程序，确保使用 UTF-8 编码写入

### 问题 4：重复文章判断困难
**原因：** 同一研究多个版本
**解决：** 对比标题关键词、研究机构、核心技术描述
